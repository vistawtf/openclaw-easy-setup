OpenClaw: from zero to AI assistant

Follow along during the workshop. Commands are ready to copy.

0
Why: what OpenClaw is and what you'll build
3 min

ChatGPT forgets you after every conversation. OpenClaw gives a model persistent memory, tools, and communication channels — turning a chatbot into an assistant that actually knows you. By the end of this workshop, you'll have one running on Telegram.

Cost heads up OpenClaw itself is free and open source. The AI models behind it (Anthropic, OpenAI) cost money. For moderate personal use, expect roughly $10–30/month.
1
Your environment: where to run OpenClaw
3 min

OpenClaw runs as a persistent process — it needs a machine that stays on. Your AI assistant has access to files, shell, and network on whatever machine it runs on, so where you run it matters.

For professional use, a dedicated VPS is the right call. Three reasons:

  • Security — your personal files, work accounts, and browser sessions stay separate from your assistant's workspace
  • Availability — crons and heartbeats stop when your laptop sleeps. A server runs 24/7.
  • Separation of concerns — the agent's workspace is isolated and easier to back up and harden

OpenClaw itself is lightweight — 1 CPU, 1GB RAM, 10GB disk is enough. The AI models run remotely (Anthropic, OpenAI), so your server just needs to stay connected.

Software prerequisites
2 min

Make sure you have these before running anything.

  • Ubuntu, macOS, or Windows with WSL2
  • An Anthropic account with an API key — console.anthropic.com
  • A Telegram account (phone + desktop app)
  • Node.js 18+ (we'll install it in the next section if you don't have it)
3
Install OpenClaw and connect Telegram
12 min

Install OpenClaw, create your Telegram bot, and run the setup wizard. By the end of this section, your assistant will be live on Telegram.

1. Install system dependencies (Ubuntu/WSL2 only)

sudo apt-get update && sudo apt-get install -y curl git build-essential

2. Install Node.js (if needed)

node --version

If you see v18 or higher, skip the next command.

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

3. Install OpenClaw

npm install -g openclaw
openclaw --version

4. Create your Telegram bot

Do this before running the wizard — you'll need the token in the next step.

Open Telegram, go to @BotFather, and send:

/newbot

BotFather asks for a display name and a username (must end in bot). It then gives you an API token — copy it, you'll paste it in the wizard.

5. Run the setup wizard

openclaw onboard

The wizard walks you through everything. Here's what it asks:

  • Acknowledge that OpenClaw is a personal tool with broad system access
  • Quickstart or manual — quickstart is fine
  • AI provider — choose Anthropic
  • Auth method — Personal Access Token if you have a Claude.ai subscription; API key from console.anthropic.com otherwise
  • Model — Claude Sonnet is a good default
  • Channel — select Telegram, paste the bot token from BotFather
  • Skills and hooks — skip both

6. Verify the gateway is running

openclaw health

If the gateway isn't running (health shows an error), start it manually:

nohup openclaw gateway &

This is common on VPS setups where systemd user services aren't available.

7. Pair your Telegram account

Open Telegram, find your bot, and send it any message. OpenClaw will respond with a pairing command — something like:

openclaw directory approve --channel telegram --id 123456789

Copy that exact command from Telegram and run it in your terminal. Once approved, send another message — you should get a real reply from your assistant.

7. Get your chat ID

openclaw directory self --channel telegram

Save this number — you'll need it when setting up crons later. Add it to TOOLS.md.

End state
  • OpenClaw installed (openclaw --version shows a number)
  • Gateway running (openclaw health is green)
  • Bot created in Telegram
  • Sent a test message and got a reply
  • Chat ID saved in TOOLS.md
If something breaks Run openclaw doctor — it checks and fixes common issues. If the bot doesn't respond, verify the gateway is running with openclaw health.

Your workspace: the files that matter
10 min

Your workspace is a folder (usually ~/clawd/) containing plain text files. Your assistant reads them at the start of every conversation. This is what gives it memory, personality, and context about you.

Back up first If you already have files in ~/clawd/, copying the starter pack will overwrite any files with the same names. Back up first: cp -r ~/clawd/ ~/clawd-backup/ 
clawd/
├── IDENTITY.md   ← Name and vibe (6 lines)
├── SOUL.md       ← Personality and tone
├── USER.md       ← About you: timezone, language, priorities
├── AGENTS.md     ← The rulebook: what to do and how
├── TOOLS.md      ← Your configured services
└── memory/
    └── MEMORY.md ← Long-term memory (starts empty)
Your turn — open and edit these now
  • IDENTITY.md — give your assistant a name
  • USER.md — set your timezone and preferred language
  • SOUL.md — read through it, adjust anything that doesn't feel right
  • AGENTS.md — review the rules, no need to change much yet
The most important file SOUL.md tells your assistant how not to sound like a generic AI. Without it, you have ChatGPT with memory. With it, you have an assistant with an actual personality.

Your first conversation: make sure it works
3 min

Before going further, verify the whole pipeline works. Send these four messages to your bot in Telegram.

Send these in Telegram
  • "What's your name?" — should match IDENTITY.md
  • "What timezone am I in?" — should match USER.md
  • "Give me your honest opinion: is it worth learning to code in 2026?" — should have a real take, not "it depends"
  • "Remember this: I prefer bullet points over paragraphs." Then: "What do you know about my preferences?"
If the personality feels generic Go back to SOUL.md and strengthen the tone instructions. The defaults are a starting point, meant to be edited.
5
Memory: how your AI remembers
5 min

Your assistant doesn't inherently remember yesterday. The files in your workspace are its memory. When it "remembers" something, it wrote to a file. When it recalls something, it read one.

Three layers
  • Bootstrap files (SOUL.md, USER.md, etc.) — loaded every session, always available
  • Daily logs (memory/YYYY-MM-DD.md) — one file per day, created automatically
  • MEMORY.md — long-term highlights. Starts empty, grows naturally over weeks
Your turn
  • Check that memory/ directory exists in your workspace
  • Check that memory/MEMORY.md exists (it can be empty)
  • After your first real conversation, check that a daily log was created

Skills: ready-made capabilities
4 min

Skills are specialized instruction files your assistant loads automatically when a task matches. Ask it to summarize a video, it loads summarize. Ask it to review a pull request, it loads github. You don't manage this manually.

See what's installed

openclaw skills list

Get details on a specific skill

openclaw skills info summarize
Good ones to start with
  • summarize — paste a URL, get a summary
  • github — issues, PRs, CI status (if you write code)
  • weather — useful for morning briefing crons
  • humanizer — makes AI-sounding text more natural
Want the full breakdown? For a detailed look at the bundled skills and how to create your own, see the Skills Guide.

Crons: your AI works while you sleep
6 min

Crons are scheduled tasks. Tell your assistant "every morning at 8am, send me a briefing on Telegram" and it runs in the background, no computer needed. This is where OpenClaw goes from chat assistant to personal system.

Test cron (fires in 1 minute, then self-destructs)

openclaw cron create \
  --name "Test Cron" \
  --at "+1m" \
  --message "Say hello! This is a test cron." \
  --to telegram:YOUR_CHAT_ID \
  --announce \
  --wake now \
  --delete-after-run

Real cron — morning briefing

openclaw cron create \
  --name "Good Morning" \
  --cron "0 12 * * *" \
  --message "Give me a brief morning briefing: what day it is, a motivational thought, and one thing I should focus on today." \
  --to telegram:YOUR_CHAT_ID \
  --announce \
  --wake now
Time zones Cron times are in UTC by default. If you're in UTC-4 (Eastern), 0 12 * * * fires at 8am your time. Use --tz America/New_York to set your timezone explicitly. Use crontab.guru to build expressions.

Manage your crons

openclaw cron list
openclaw cron delete JOB_ID
Your turn
  • Run the test cron and confirm it arrives on Telegram
  • Create your real first cron (morning briefing or similar)
  • Keep it to 15 active crons max
If something breaks If the cron runs but nothing arrives: check the chat ID and make sure --announce is there. If the cron doesn't run at all: openclaw health to check the gateway.

Security basics: two things to get right
3 min

Security goes deep, but today you only need two rules. Everything else is in SECURITY.md when you're ready for it.

Rule 1: API keys don't go in your workspace Your workspace files are things you might back up or share someday. API keys live in ~/.config/<service>/ or in OpenClaw's own config at ~/.openclaw/. Never in SOUL.md, TOOLS.md, or any workspace file. Automated bots scan GitHub for leaked keys within minutes.
Rule 2: Telegram allowlist We set this up in section 4. Without it, anyone who finds your bot's username can talk to your AI. If you skipped it, go back and run openclaw configure --section channels.
Want to go deeper? Run openclaw security audit anytime to check your setup. SECURITY.md in the starter pack has the full guide: file permissions, secrets management, tool policies, Docker sandboxing.
ClawdStrike — security audit skill ClawdStrike is a skill built by Cantina that scans your OpenClaw setup for common security issues. Worth running once you're up and running. 
npx skills add https://github.com/cantinaxyz/clawdstrike --skill clawdstrike
Move it to .openclaw/workspace/skills and tell your assistant to run the ClawdStrike skill.
Your turn
  • Confirm no API keys are inside your workspace files
  • Confirm Telegram DM policy is set to pairing

Your first real day: putting it to work
4 min

Everything's set up. Try these in Telegram right now — they all work with just what we built today.

Try these in Telegram
  • "Remind me tomorrow at 9am to send the weekly report" — watch it create a cron automatically
  • "Based on what you know about me, suggest three weekend project ideas" — should reflect your context from USER.md
  • "Remember: my boss's name is Sarah, team standup is at 10am EST" — then check your memory files
The growth loop Update SOUL.md when something about the tone bothers you. Add rules to AGENTS.md when you want a specific behavior. Let the memory folder grow naturally. The more context it has, the more useful it gets.

What comes next
2 min

None of this is urgent today. But knowing it exists saves time later.

  • More channels — Discord, WhatsApp, Slack. Same pattern as Telegram.
  • Notion integration — create and manage tasks from Telegram.
  • Brave Search API (free) — lets your assistant search the web.
  • Custom skills — when you repeat the same instructions, write a SKILL.md file.
  • Subagents — delegate long tasks while your main assistant stays free for urgent things.
  • Security hardening — SECURITY.md in the starter pack. Run openclaw security audit when ready.
