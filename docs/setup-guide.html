Setup Guide

Set up once. Works forever. Gets better every night.

~57 minutes Ubuntu / macOS / WSL2 End state: AI in Telegram with memory + crons
0
Why: what OpenClaw is and what you'll build

ChatGPT forgets you after every conversation. OpenClaw gives a model persistent memory, tools, and communication channels — turning a chatbot into an assistant that actually knows you. By the end of this guide, you'll have one running on Telegram.

Cost heads up OpenClaw itself is free and open source. The AI models behind it cost money — providers like Anthropic and OpenAI charge per use. For moderate personal use, expect roughly $10–30/month.
1
Your environment: where to run OpenClaw

OpenClaw runs as a persistent process — it needs a machine that stays on. Your AI assistant has access to files, shell, and network on whatever machine it runs on, so where you run it matters.

For professional use, a dedicated VPS is the right call. Three reasons:

  • Security — your personal files, work accounts, and browser sessions stay separate from your assistant's workspace
  • Availability — crons and heartbeats stop when your laptop sleeps. A server runs 24/7.
  • Separation of concerns — the agent's workspace is isolated and easier to back up and harden

OpenClaw itself is lightweight — 1 CPU, 1GB RAM, 10GB disk is enough. The AI models run remotely on your chosen provider's servers, so your machine just needs to stay connected.

Software prerequisites

Make sure you have these before running anything.

  • Ubuntu, macOS, or Windows with WSL2
  • An account with an AI provider — we recommend Anthropic (Claude), but OpenAI and others work too
  • A Telegram account (phone + desktop app)
  • Node.js 18+ (we'll install it in the next section if you don't have it)
3
Install OpenClaw

Install system dependencies and OpenClaw itself. The starter pack comes next — install it before running the setup wizard.

1. Install system dependencies (Ubuntu/Debian only)

sudo apt-get update && sudo apt-get install -y curl git build-essential

2. Install Node.js (if needed)

node --version

If you see v18 or higher, skip the next command.

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

3. Install OpenClaw

npm install -g openclaw
openclaw --version
4
Install the starter pack

Install the starter pack before running the setup wizard — OpenClaw will detect it and use it as your workspace automatically.

cd ~ && git clone https://github.com/vistawtf/openclaw-easy-setup.git
mkdir -p ~/.openclaw/workspace/
cp -r openclaw-easy-setup/starter-pack/. ~/.openclaw/workspace/
5
Run the setup wizard

The wizard configures your AI provider, connects Telegram, and starts the gateway. Run it now that the starter pack is in place.

1. Create your Telegram bot first

Open Telegram, go to @BotFather, and send /newbot. It will ask for a display name and a username (must end in bot). Copy the token it gives you — you'll paste it in the wizard.

2. Run the wizard

openclaw onboard

What it asks:

  • Acknowledge that OpenClaw is a personal tool with broad system access
  • Quickstart or manual — quickstart is fine
  • AI provider — pick one from the list (Claude is a good starting point)
  • Auth method — follow the on-screen instructions for your chosen provider
  • Model — pick the default or Claude Sonnet if you went with Anthropic
  • Channel — select Telegram, paste the bot token from BotFather
  • Skills and hooks — skip both

3. Verify the gateway is running

openclaw health

If the gateway isn't running, start it manually:

nohup openclaw gateway &

This is common on VPS setups where systemd user services aren't available.

If something breaks Run openclaw doctor — it checks and fixes common issues automatically.
6
Meet your assistant

Almost there. First, approve the Telegram pairing request — then send your bot a message and the onboarding begins.

1. Approve the pairing request

After the wizard finishes, your terminal will show a pairing code. Run this to approve it:

openclaw pairing approve telegram XXXXXXXX

Replace XXXXXXXX with the code shown in your terminal.

2. Send your bot a message

Open Telegram, find your bot, and send it any message. Because BOOTSTRAP.md is in your workspace, your assistant will kick off a short onboarding conversation before doing anything else.

What happens

Your assistant reads BOOTSTRAP.md and guides you through personalizing your workspace. It asks one question at a time:

  • What to call it — give it a name
  • Your name and role — so it knows who it's helping
  • Timezone and working hours — so it doesn't ping you at 3am
  • Your language — it will match you going forward
  • Your current priorities — what's taking up your headspace
  • Feedback style — blunt or collaborative?
  • Hard rules — anything it should never do
  • Services — Notion, GitHub, or others you use

At the end it sets up the Auto-Improve loop (see next section) and restarts itself with your new config.

Security note Never send API keys or passwords through chat. If your assistant needs you to connect a service, it will give you a terminal command to run — not ask you to paste secrets in the conversation.
7
Auto-improve: it gets better while you sleep

The starter pack includes a nightly self-improvement loop. Every night at 4am UTC, your assistant picks one friction point from the day, fixes it, and logs what it did.

What it does each night

  • Picks one small friction point and fixes it autonomously
  • Cleans up stale files, empty docs, scattered notes
  • Logs what it did in memory/auto-improve-log.md

Weekly report (every Sunday)

On Sundays it reads the week's memory files, finds patterns in how you work, and generates a short report in memory/auto-improve-reports/. You'll get it in Telegram.

Managing the cron

openclaw cron list          # see all scheduled tasks
openclaw cron delete <id>  # remove auto-improve if you want to pause it

Auto-Improve was set up during the BOOTSTRAP conversation. It runs automatically. You can adjust the schedule or prompt anytime by editing the cron via openclaw cron list.

8
Memory: how your AI remembers

Your assistant doesn't inherently remember yesterday. The files in your workspace are its memory. When it "remembers" something, it wrote to a file. When it recalls something, it read one.

Three layers

  • Bootstrap files (SOUL.md, USER.md, etc.) — loaded every session, always available
  • Daily logs (memory/YYYY-MM-DD.md) — one file per day, created automatically
  • MEMORY.md — long-term highlights. Starts empty, grows naturally over weeks

Verify it's working

  • Check that memory/ directory exists in your workspace
  • Check that memory/MEMORY.md exists (it can be empty)
  • After your first real conversation, check that a daily log was created

Skills: ready-made capabilities

Skills are specialized instruction files your assistant loads automatically when a task matches. Ask it to summarize a video, it loads summarize. Ask it to review a pull request, it loads github. You don't manage this manually.

What's already active

The starter pack ships with three skills already installed. No setup needed — they're live from the moment BOOTSTRAP finishes.

  • auto-improve — nightly self-improvement loop (covered in the previous section)
  • humanizer — rewrites AI-sounding text to sound like a person. Useful for public-facing content.
  • skill-creator — guides you through building your own skills for specific workflows.

See what's installed

openclaw skills list

Good ones to add next

  • summarize — paste a URL, get a summary
  • github — issues, PRs, CI status (if you write code)
  • weather — useful for morning briefing crons

For a detailed look at the bundled skills and how to create your own, see the Skills Guide.

Crons: your AI works while you sleep

Crons are scheduled tasks. Tell your assistant "every morning at 8am, send me a briefing on Telegram" and it runs in the background, no computer needed. This is where OpenClaw goes from chat assistant to personal system.

Test cron (fires in 1 minute, then self-destructs)

openclaw cron create \
  --name "Test Cron" \
  --at "+1m" \
  --message "Say hello! This is a test cron." \
  --to telegram:YOUR_CHAT_ID \
  --announce \
  --wake now \
  --delete-after-run

Real cron — morning briefing

openclaw cron create \
  --name "Good Morning" \
  --cron "0 12 * * *" \
  --message "Give me a brief morning briefing: what day it is, a motivational thought, and one thing I should focus on today." \
  --to telegram:YOUR_CHAT_ID \
  --announce \
  --wake now

Cron times are in UTC by default. If you're in UTC-4 (Eastern), 0 12 * * * fires at 8am your time. Use --tz America/New_York to set your timezone explicitly. Use crontab.guru to build expressions.

Manage your crons

openclaw cron list
openclaw cron delete JOB_ID

Try it out

  • Run the test cron and confirm it arrives on Telegram
  • Create your real first cron (morning briefing or similar)
  • Keep it to 15 active crons max
If something breaks If the cron runs but nothing arrives: check the chat ID and make sure --announce is there. If the cron doesn't run at all: openclaw health to check the gateway.

Security basics: two things to get right

Security goes deep, but today you only need two rules. Everything else is in SECURITY.md when you're ready for it.

  1. API keys don't go in your workspace. Your workspace files are things you might back up or share someday. API keys live in ~/.config/<service>/ or in OpenClaw's own config at ~/.openclaw/. Never in SOUL.md, TOOLS.md, or any workspace file. Automated bots scan GitHub for leaked keys within minutes.
  2. Set up a Telegram allowlist. Without it, anyone who finds your bot's username can talk to your AI. Run openclaw configure --section channels to set it up.

Going deeper

Run openclaw security audit anytime to check your setup. SECURITY.md in the starter pack has the full guide: file permissions, secrets management, tool policies, Docker sandboxing.

ClawdStrike is a security audit skill built by Cantina that scans your OpenClaw setup for common issues. Worth running once you're up and running:

npx skills add https://github.com/cantinaxyz/clawdstrike --skill clawdstrike

Move it to .openclaw/workspace/skills and tell your assistant to run the ClawdStrike skill.

Verify your setup

  • Confirm no API keys are inside your workspace files
  • Confirm Telegram DM policy is set to pairing

Your first real day: putting it to work

Everything's set up. Try these in Telegram right now — they all work with just what we built today.

Try these in Telegram

  • "Remind me tomorrow at 9am to send the weekly report" — watch it create a cron automatically
  • "Based on what you know about me, suggest three weekend project ideas" — should reflect your context from USER.md
  • "Remember: my boss's name is Sarah, team standup is at 10am EST" — then check your memory files

The growth loop

Update SOUL.md when something about the tone bothers you. Add rules to AGENTS.md when you want a specific behavior. Let the memory folder grow naturally. The more context it has, the more useful it gets.

What comes next

None of this is urgent today. But knowing it exists saves time later.

  • More channels — Discord, WhatsApp, Slack. Same pattern as Telegram.
