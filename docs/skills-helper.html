What are skills?

Modular capabilities your AI loads automatically

Skills are specialized instruction sets that give your AI knowledge it wouldn't have otherwise. Each skill is a markdown file (SKILL.md) that lives in your skills/ directory.

How they work

  1. OpenClaw scans your skills/ directory at startup
  2. It reads each skill's name and description (just the metadata, not the full file)
  3. When you ask your AI something that matches a skill's description, OpenClaw loads the full skill instructions into context
  4. Your AI follows those instructions to handle the task

You don't need to tell your AI to "use" a skill. It happens automatically based on what you ask.

Skills are lazy-loaded Only the skill that matches your request gets loaded. Having 20 skills installed doesn't slow anything down or use extra tokens until one actually triggers.

Skills included in this pack

Three skills ready to use out of the box
Skill What it does When it triggers
auto-improve Runs overnight: fixes one friction point, tidies your workspace, generates weekly learning reports Via a scheduled cron job (set up during bootstrap or manually)
humanizer Removes AI writing patterns from text: em dash overuse, significance inflation, rule-of-three, sycophantic tone, and 20+ other tells When you ask to edit, review, or "humanize" text. Also useful for blog posts, emails, and public content
skill-creator Guides you through building your own custom skills with proper structure, frontmatter, and bundled resources When you say "create a skill", "build a skill", or want to package domain-specific knowledge

auto-improve

This is the self-improvement loop. It runs nightly at 4am UTC via a cron job set up during bootstrap. Each run, it picks one friction point from the day's interactions — a workflow that took too many steps, a config rule that was missing, a pattern the AI keeps getting wrong — and fixes it autonomously.

Changes are logged in memory/auto-improve-log.md. Every Sunday it generates a weekly report in memory/auto-improve-reports/ summarizing what changed and what's still worth fixing.

The key thing: it improves based on your specific usage, not a generic template. The longer you use OpenClaw, the more precisely it calibrates to your setup. In week one it's useful. In month six it's operating on months of learned patterns specific to you.

The bootstrap wizard offers to set the cron up for you. If you skipped it, the exact cron command is in skills/auto-improve/SKILL.md.

humanizer

Rewrites AI-sounding text to read like a person wrote it. Based on Wikipedia's "Signs of AI writing" guide — which is the most comprehensive public catalog of these patterns.

What it detects and fixes: em dash overuse (replacing with commas or periods), the "rule of three" (lists of exactly three that don't need to be), significance inflation ("pivotal", "transformative", "testament to"), negative parallelisms ("it's not just X, it's Y" — just say Y), filler phrases ("in order to", "due to the fact that"), hedging preambles ("it's worth noting that"), copula avoidance ("serves as", "functions as" instead of just "is"), and synonym cycling.

Use it before publishing anything public: articles, emails, website copy, social posts. The assistant applies it automatically to public-facing content — you can also trigger it manually by saying "humanize this" or "make this sound less AI-written".

skill-creator

Guides you through building your own custom skills. Skills are markdown files with a frontmatter block (name and description) plus plain-language instructions — the AI reads them when the task matches.

The skill-creator covers the full process: figuring out what the skill should do and what triggers it, planning any supporting resources (reference files, scripts, example outputs), writing the SKILL.md with the right structure, and testing that it triggers reliably.

Best time to use it: after you've been working with OpenClaw for a few weeks and notice you keep explaining the same context over and over. That repeated context is a skill waiting to be written. Tell your AI "I want to create a new skill for [X]" and it takes it from there.

Skill directory structure Each skill lives in its own folder under skills/: 
skills/
├── auto-improve/
│   └── SKILL.md
├── humanizer/
│   └── SKILL.md
└── skill-creator/
    └── SKILL.md

Finding more skills

The ClawHub marketplace and community

The ClawHub marketplace is where the community shares skills. Browse by category or search for what you need.

Installing a skill from ClawHub

The easiest way: tell your assistant what you need. For example:

  • "I want a skill for summarizing web pages and YouTube videos"
  • "Install the github skill"
  • "I need help with spreadsheets — is there a skill for that?"

Your AI will help you find and install the right skill. You can also paste a ClawHub skill link directly into chat.

Creating your own skills

Package your domain knowledge for reuse

The best skills come from noticing what you repeat. If you keep explaining the same context, running the same workflow, or referencing the same docs, that's a skill waiting to be written.

Quick overview

  1. Create a folder in skills/ with your skill name
  2. Add a SKILL.md with YAML frontmatter (name and description) and markdown instructions
  3. Optionally add scripts/, references/, or assets/ folders for supporting files

Minimal example

---
name: daily-standup
description: Generate a daily standup update from recent work logs and git activity. Use when the user asks for a standup, daily update, or status report.
---

# Daily Standup

Read the last 24 hours of memory files and git logs.
Format as:
- **Yesterday:** what was completed
- **Today:** what's planned
- **Blockers:** anything stuck

Keep it under 5 lines total.

The bundled skill-creator skill walks you through the full process with best practices for structure, progressive disclosure, and testing. Just tell your AI: "I want to create a new skill."

Start simple Your first skill should be 20-50 lines. A frontmatter block plus instructions. Don't over-engineer it. You can always add scripts and references later when you see what's missing.
